# توثيق حالات الاستخدام (Use‑Case Descriptions)

هذا المستند يقدّم توصيفًا تفصيليًا لكل حالة استخدام في مشروع التجارة الإلكترونية. يمكن استعماله في وثائق المشروع أو في أدوات تتبع المتطلبات.

---

## 1️⃣ Register (تسجيل حساب جديد)
- **الجهة (Actor)**: **العميل**
- **الهدف**: إنشاء حساب شخصي جديد يسمح للعميل بإجراء عمليات الشراء والحصول على سجل الطلبات.
- **الافتراضات المسبقة**: العميل ليس مسجلاً مسبقًا في النظام.
- **الخطوات الأساسية**:
  1. العميل يضغط على زر "تسجيل" في واجهة التسجيل.
  2. يملأ النموذج بالبيانات المطلوبة: اسم المستخدم، البريد الإلكتروني، كلمة المرور.
  3. يضغط على زر "إنشاء حساب".
  4. يخزن الخادم البيانات في جدول `users` (أو في ملف JSON إذا لم يتوفر PostgreSQL).
  5. تُرسل ردّ إيجابي للعميل مع رسالة تأكيد.
- **النتيجة**: حساب جديد يُضاف إلى قاعدة البيانات ويصبح بإمكان العميل تسجيل الدخول.

---

## 2️⃣ Login (تسجيل الدخول)
- **الجهة**: **العميل**
- **الهدف**: تمكين العميل من الوصول إلى حسابه لتصفح الطلبات وإدارة السلة.
- **الافتراضات المسبقة**: حساب المستخدم موجود في قاعدة البيانات.
- **الخطوات الأساسية**:
  1. يفتح العميل صفحة الدخول ويُدخل اسم المستخدم أو البريد الإلكتروني وكلمة المرور.
  2. الخادم يتحقق من صحة البيانات بإجراء استعلام على جدول `users` أو ملف `users.json`.
  3. إذا كانت البيانات صحيحة يُعيد الخادم ردًّا يضم معلومات المستخدم (دون كلمة المرور).
  4. يُخزن المتصفح البيانات في `localStorage` / `sessionStorage` أو يُرسل كـ JWT (حسب التنفيذ).
- **النتيجة**: يُصبح المستخدم مُوثقًا ويستطيع الوصول إلى الموارد المحمية.

---

## 3️⃣ Browse Products (تصفح المنتجات)
- **الجهة**: **العميل**
- **الهدف**: عرض قائمة المنتجات المتاحة.
- **الخطوات الأساسية**:
  1. يرسل المتصفح طلب GET إلى `/api/products`.
  2. الخادم يسترجع جميع المنتجات من جدول `products` أو ملف `products.json`.
  3. تُعاد البيانات على هيئة JSON وتُعرض على الصفحة.
- **ملاحظة**: يستخدم الخادم `DBHelper.getProducts()` الذي يُعيد الحقول `id, title, price, image, type, category_id, sizes, description, quantity, discount`.

---

## 4️⃣ Search Products (بحث عن منتجات)
- **الجهة**: **العميل**
- **الهدف**: تمكين العميل من العثور على منتجات بناءً على كلمة مفتاحية.
- **الخطوات الأساسية**:
  1. يكتب العميل نصًا في حقل البحث.
  2. يُرسل المتصفح طلب GET إلى `/api/products?query=الكلمة` (يمكنك إضافة معلمة `query` في النهاية).
  3. الخادم يفلتر النتائج إما في قاعدة البيانات (`WHERE title ILIKE $1`) أو في ملف JSON باستخدام `.filter`.
  4. تُعاد النتائج وتُعرض.

---

## 5️⃣ View Product Details (عرض تفاصيل المنتج)
- **الجهة**: **العميل**
- **الهدف**: إظهار صفحة تفصيلية للمنتج المختار.
- **الخطوات الأساسية**:
  1. يضغط العميل على بطاقة المنتج.
  2. المتصفح يطلب `/api/products/:id` (ليس مُعرّفًا في الكود الآن؛ يمكن إضافته).
  3. الخادم يجلب السجل وفق `id` ويعيد JSON يحتوي على جميع الحقول.
  4. تُعرض معلومات مثل الصور، الوصف، الأحجام المتاحة، الخصومات.

---

## 6️⃣ Add to Cart (إضافة إلى السلة)
- **الجهة**: **العميل**
- **الهدف**: حفظ منتج مؤقتًا في سلة الشراء للمستخدم.
- **الخطوات الأساسية**:
  1. يضغط العميل على زر "إضافة إلى السلة" مع اختيار الحجم والكمية.
  2. المتصفح يرسل طلب POST إلى `/api/cart` مع جسم JSON:
     ```json
     { "username": "<اسم_المستخدم>", "items": [{ "id": <product_id>, "quantity": 1, "size": "M" }] }
     ```
  3. الخادم يستدعي `DBHelper.saveCart(username, items)`، والذي يتحقق من وجود المنتج في جدول `products` قبل الإدخال لتجنّب الأخطاء.
  4. تُرجع استجابة نجاح.
- **النتيجة**: يُضيف العنصر إلى جدول `cart_items` (PostgreSQL) أو إلى ملف `carts.json`.

---

## 7️⃣ Manage Cart (إدارة السلة)
- **الجهة**: **العميل**
- **الهدف**: عرض، تعديل أو حذف عناصر السلة قبل إتمام الشراء.
- **الخطوات الأساسية**:
  1. المتصفح يرسل طلب GET إلى `/api/cart?username=<اسم>` لاسترجاع محتوى السلة.
  2. لتحديث كمية أو حجم، يُرسل طلب PUT/POST إلى نفس المسار مع قائمة العناصر المعدّلة.
  3. لحذف عنصر، يُرسل طلب DELETE مع `product_id` داخل جسم الطلب.
  4. الخادم يحدّث `cart_items` أو `carts.json` وفقًا للبيانات الجديدة.

---

## 8️⃣ Checkout (إتمام العملية)
- **الجهة**: **العميل**
- **الهدف**: إنشاء طلب (order) نهائي، تخزينه، وتحديث مخزون المنتجات.
- **الخطوات الأساسية**:
  1. يضغط العميل على زر "متابعة للشراء".
  2. المتصفح يرسل طلب POST إلى `/api/orders` مع جسم JSON يضم:
     ```json
     {
       "name": "<الاسم>",
       "email": "<البريد>",
       "phone": "<رقم الهاتف>",
       "address": "<العنوان>",
       "items": [ { "id": <product_id>, "quantity": 2, "size": "L", "price": 180 } ],
       "status": "قيد التحضير",
       "total": 360,
       "payment_method": "كاش عند الاستلام"
     }
     ```
  3. الخادم ينفّذ `DBHelper.createOrder(newOrder)`:
     - يُولّد معرف طلب فريد (إذا لم يُحدَّد).
     - يُدخل السطر في جدول `orders`.
     - يُدرج عناصر الطلب في جدول `order_items` بعد التحقق من وجود المنتج.
     - يُحدّث كمية المخزون (`quantity`) لكل منتج لتجنب overselling.
  4. يُعيد الخادم ردًّا `{ success: true, order: <orderObject> }`.
- **النتيجة**: يُنشأ سجل طلب جديد وتُفرغ السلة للعميل.

---

## 9️⃣ View Order History (عرض تاريخ الطلبات)
- **الجهة**: **العميل**
- **الهدف**: تمكين العميل من مراجعة طلباته السابقة.
- **الخطوات الأساسية**:
  1. المتصفح يرسل طلب GET إلى `/api/orders?username=<اسم>`.
  2. الخادم يستدعي `DBHelper.getOrders(username)` ويعيد جميع الطلبات المرتبطة بالمستخدم.
  3. تُظهر الواجهة قائمة الطلبات مع تفاصيل الحالة، المجموع، وتاريخ الإنشاء.

---

## 🔟 Update Profile (تحديث الملف الشخصي)
- **الجهة**: **العميل**
- **الهدف**: السماح للعميل بتعديل معلوماته (اسم، بريد، كلمة مرور).
- **الخطوات الأساسية**:
  1. يرسل المتصفح طلب PUT إلى `/api/users/:id` (لم يُنفّذ بعد، يمكن إضافته).
  2. يُحدّث الخادم السجل في جدول `users` أو ملف `users.json`.
  3. تُعيد الاستجابة بيانات المستخدم المحدثة.

---

## 🛠️ Manage Products (إدارة المنتجات)
- **الجهة**: **المسؤول**
- **الهدف**: إضافة، تعديل أو حذف المنتجات.
- **الخطوات الأساسية**:
  1. **إضافة**: POST إلى `/api/products` مع تفاصيل المنتج.
  2. **تعديل**: POST إلى `/api/products` مرة أخرى مع `id` موجود.
  3. **حذف**: DELETE إلى `/api/products?id=<product_id>`.
  4. يتم استدعاء `DBHelper.saveProduct` أو `DBHelper.deleteProduct` وفقًا للبيانات.

---

## 📦 Manage Orders (إدارة الطلبات)
- **الجهة**: **المسؤول**
- **الهدف**: مراقبة الطلبات وتحديث حالة الشحن أو الإلغاء.
- **الخطوات الأساسية**:
  1. GET إلى `/api/orders` للحصول على جميع الطلبات.
  2. POST إلى `/api/orders` مع `action: "updateStatus"` لتغيير حالة الطلب، أو `action: "updatePaymentStatus"` لتحديث حالة الدفع.

---

## 📊 Generate Sales Reports (تقارير المبيعات)
- **الجهة**: **المسؤول**
- **الهدف**: استخراج إحصائيات المبيعات (يمكن توسيعه لاحقًا بإنشاء نقطة API مخصّصة).

---

## 📦 Update Inventory (تحديث المخزون)
- **الجهة**: **المسؤول**
- **الهدف**: تعديل كمية المخزون يدويًا إذا لزم الأمر.
- **الخطوات الأساسية**:
  1. يمكن تعديل المنتج عبر **Manage Products** حيث يُحدَّث حقل `quantity`.

---

## 👥 View All Users (عرض جميع المستخدمين)
- **الجهة**: **المسؤول**
- **الهدف**: استعراض قائمة جميع العملاء والمسؤولين.

---

### ملاحظة تقنية
* جميع نقاط API تُرجع رؤوس **Cache-Control** مع `no-store` لضمان جلب البيانات مباشرة من قاعدة البيانات وتجنّب الاعتماد على التخزين المؤقت للمتصفح.
* عند تشغيل المشروع على بيئة PostgreSQL يتم تفعيل **التحقق من وجود المنتج** قبل إضافة أو تعديل السلة أو الطلب لتفادي الأخطاء الناتجة عن بيانات قديمة في الـ cache.

---

## كيف تستخدم هذا الوثيق؟
- احفظه كملف `USECASE_DESCRIPTIONS.md` داخل مجلد المشروع (مثلاً `docs/` أو `public/`).
- يمكنك ربطه مع أدوات تتبع المتطلبات (Jira, Trello) عن طريق نسخ اسم كل حالة استخدام كـ **Issue**.
- عند إضافة ميزات جديدة، قم بتحديث توصيف الحالة أو إضافة حالة جديدة مماثلة.
